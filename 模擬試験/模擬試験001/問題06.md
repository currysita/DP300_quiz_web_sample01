## 質問 6/50

複数のデータベースを含む Microsoft SQL Server 2022 インスタンスがあります。
インスタンスのディザスター リカバリー ソリューションを計画しています。
すべてのデータベースを任意の時点に復元できるようにする必要があります。
ソリューションにデータベースごとに含める必要がある 3 つのアクションはどれですか? 各正解は解答の一部を示します。

### 選択肢

- [正解] 完全なデータベース バックアップを含める
- [正解] ログ バックアップを含める
- 一括ログ復旧モデルを設定する
- "エラー時に続行" オプションを設定する
- "コピー専用" バックアップ オプションを設定する
- [正解] 完全復旧モデを設定する

### 解説

完全なデータベース バックアップとトランザクション ログ データベース バックアップの両方が必要です。 データベースは、完全復旧モデルに設定する必要があります。
一括復旧モデルでは特定の時点への復旧を設定できない場合があるため、データベースを一括復旧モデルに設定するべきではありません。
"コピー専用" オプションは、従来のバックアップとは別に使用されるバックアップを実行するために使用されます。 これは、ディザスター リカバリー戦略を構築する場合には有用ではありません。
"エラー時に続行" オプションは、障害発生時にバックアップを実行する必要がある場合に使用されます。
