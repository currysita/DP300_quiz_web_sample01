## 質問 12/50

Sales という名前のテーブルからデータを選択する動的 SQL コードを実行するストアド プロシージャを作成します。
すべてのユーザーは、このストアド プロシージャに対する実行アクセス許可を持ちます。
ユーザーがエラーなしでプロシージャを実行できるようにするには、どうすればよいですか?

### 選択肢

- WITH RECOMPILE 句をストアド プロシージャに追加します。
- ストアド プロシージャに EXECUTE AS [owner] 句を追加します。
- [正解] ユーザーの Sales テーブルに SELECT 権限を追加します。

### 解説

動的 SQL コードを実行するには、呼び出し元がコード内で呼び出されるオブジェクトに必要なすべてのアクセス許可を持っている必要があるため、Sales テーブルに SELECT 権限を追加する必要があります。
EXECUTE AS [owner] 句または WITH RECOMPILE 句を使用しても、呼び出し元がコードで呼び出されるオブジェクトに対して必要なすべてのアクセス許可を持っているわけではありません。
