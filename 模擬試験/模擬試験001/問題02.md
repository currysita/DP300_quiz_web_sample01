## 質問 2/50

SQLVM1 という名前のAzure仮想マシンにMicrosoft SQL Server インスタンスがインストールされている。
バッファー キャッシュやバッチ要求など、SQL Server エンジン カウンターを 1 秒あたりに分析する必要があります。
これらのメトリックを収集するために何を使用する必要がありますか?

### 選択肢

- Azure アクティビティ ログ
- Azure ポータルでのパフォーマンス分析情報のクエリ
- [正解] SQLVM1 のWindows パフォーマンス モニター (PerfMon)
- Azure Monitor の Microsoft.Compute/virtualMachines リソースに関するメトリック

### 解説

目的：
3.1 リソースのアクティビティとパフォーマンスを監視する

              この項目のテスト内容:
パフォーマンス メトリックのソースを決定する

              追加読書:


              パフォーマンス監視ツールについて説明する

              まとめ

              データベースのパフォーマンスを監視する
 Azure SQL Database および Azure SQL Managed Instance

              Monitor Azure SQL Managed Instance


              根拠：
SQLVM1 のWindows パフォーマンス モニター (PerfMon) は、バッファー キャッシュ ヒット率や 1 秒あたりのバッチ要求などのSQL Server エンジン カウンターを収集できます。これは、仮想マシン ホスト上のWindows パフォーマンス カウンター サブシステムを介して公開SQL Serverパフォーマンス オブジェクトにアクセスするためです。 Azure MonitorのMicrosoft.Compute/virtualMachines リソースのメトリックは、CPU、ディスク、ネットワーク使用量などのインフラストラクチャ メトリックのみを公開し、SQL Serverエンジン カウンターは含まれません。 Query Performance Insight は、仮想マシン上のSQL Serverではなく、Azure SQL Databaseに適用されます。 Azure アクティビティ ログには、データベース エンジンのパフォーマンス メトリックではなく、Azureのコントロール プレーン操作が記録されます。
