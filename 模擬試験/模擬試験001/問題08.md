## 質問 8/50

Azure SQL Managed Instanceにデータベースがある。
ワークロード レベルで MAXDOP 構成を制御する必要があります。 このソリューションでは、管理作業と開発作業を最小限に抑える必要があります。
どうすればよいでしょうか?

### 選択肢

- [選択・不正解] データベース クエリ レベルで、MAXDOP ヒントを設定します。
- ALTER DATABASE SCOPED CONFIGURATION を使用して、データベースの MAXDOP オプションを構成します。
- max degree of parallelism パラメーターを使用して、sp_configure ストアド プロシージャを実行します。
- [正解] Resource Governorで、MAX_DOPワークロード グループの構成を設定します。

### 解説

定義されたワークロードごとの並列処理の程度を制御できるように、Resource GovernorでMAX_DOPワークロード グループの構成を構成します。 このオプションは、CREATE WORKLOAD GROUP ステートメントで設定できます。
ALTER DATABASE SCOPED CONFIGURATION ステートメントは、ワークロードごとにではなく、データベースごとの設定を管理します。 クエリ レベルで MAXDOP ヒントを使用するには、コードを変更する必要があり、ワークロードを制御することはできません。 sp_configure ストアド プロシージャを使用すると、サーバー レベルで設定が変更され、ワークロードを制御することはできません。
