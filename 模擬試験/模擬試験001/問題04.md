## 質問 4/50

Azure SQL Managed Instanceでクエリ ストアを使用する予定です。
後退したクエリの実行プランを手動で強制できることを確認する必要があります。 解答では、最小限の特権の原則に従う必要があります。
どのデータベース ロールを自分に割り当てる必要がありますか?

### 選択肢

- データベースアクセス管理者
- [選択・不正解] db_ddladmin
- [正解] db_owner（デービーオーナー）
- データベースセキュリティ管理者 (db_securityadmin)

### 解説

db_owner固定データベース ロールのメンバーとして、最小限の特権の原則に従いながら、データベースのすべての構成およびメンテナンス アクティビティを実行できます。
db_ddladmin ロールのメンバーは、データベースに対して任意の DDL コマンドを実行できます。 このロールのメンバーは、高い権限で実行される可能性のあるコードを操作することで、権限を昇格させることができます。これらのアクションは監視する必要があります。 db_accessadmin固定データベース ロールのメンバーは、Windows ログイン、Windows グループ、およびMicrosoft SQL Server ログインのデータベースへのアクセスを追加または削除できます。 db_securityadmin 固定データベース ロールのメンバーは、カスタム ロールのみのロール メンバーシップの変更、および権限の管理を実行できます。
