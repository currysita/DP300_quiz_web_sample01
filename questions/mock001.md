# DP-300 模擬試験001

## 問題 1

新しいデータベースがAzure SQL Managed Instanceに追加されたときにイベントをキャプチャするように拡張イベント監視を構成する必要があります。どのイベントカテゴリを使用する必要がありますか？（一つ選択）

- A. 管理者
- B. 分析
- C. デバッグ
- D. 運用

<details>
<summary>解答・解説</summary>

**解答:** D

運用イベントは、問題や事象の分析および診断のために使用されます。問題や事象に応じたツールまたはタスクを起動する目的で使用できます。操作イベントの例には、データベースがサーバーに追加またはサーバーから削除される場合があります。

| 選択肢 | 説明 | 正否 |
|---|---|---|
| A. 管理者 | 管理者が実行可能な、明確に定義された一連のアクション内の問題 | ❌ |
| B. 分析 | プログラムの操作を記述し、通常はパフォーマンス調査で使用 | ❌ |
| C. デバッグ | デバッグの問題を診断するために開発者のみが使用 | ❌ |
| D. 運用 | データベースの追加・削除などの事象分析に使用 | ✅ 正解 |

</details>

## 問題 2

SQLVM1 という名前のAzure仮想マシンにMicrosoft SQL Serverインスタンスがインストールされている。バッファーキャッシュやバッチ要求など、SQL Serverエンジンカウンターを1秒あたりに分析する必要があります。これらのメトリックを収集するために何を使用する必要がありますか？（一つ選択）

- A. Azure アクティビティログ
- B. Azure ポータルでのパフォーマンス分析情報のクエリ
- C. SQLVM1 のWindows パフォーマンスモニター（PerfMon）
- D. Azure Monitor の Microsoft.Compute/virtualMachines リソースに関するメトリック

<details>
<summary>解答・解説</summary>

**解答:** C

SQLVM1 のWindows パフォーマンスモニター（PerfMon）は、バッファーキャッシュヒット率や1秒あたりのバッチ要求などのSQL Serverエンジンカウンターを収集できます。これは、仮想マシンホスト上のWindows パフォーマンスカウンターサブシステムを介して公開SQL Serverパフォーマンスオブジェクトにアクセスするためです。

| 選択肢 | 説明 | 正否 |
|---|---|---|
| A. Azure アクティビティログ | Azureのコントロールプレーン操作を記録。DBエンジンのパフォーマンスメトリックは含まない | ❌ |
| B. パフォーマンス分析情報のクエリ | Azure SQL Database向け。仮想マシン上のSQL Serverには適用されない | ❌ |
| C. PerfMon | SQL Serverエンジンカウンターを収集可能 | ✅ 正解 |
| D. Azure Monitor のVMメトリック | CPU・ディスク・ネットワークなどインフラメトリックのみ。SQL Serverエンジンカウンターは含まない | ❌ |

</details>

## 問題 3

組織はAzure SQL Databaseを使用しており、ユーザーは特にピーク時にパフォーマンスの問題を報告します。パフォーマンスの問題が発生した場合の根本原因を特定するには、リアルタイムのクエリ実行を監視する必要があります。どのアクションを実行する必要がありますか？（2つ選択）

- A. クエリストアを使用してクエリの履歴の傾向を分析します。
- B. sql_batch_completed イベントをキャプチャします。
- C. データベース監査を有効にします。
- D. 拡張イベントとフィルターセッションを使用して、システムクエリを除外します。

<details>
<summary>解答・解説</summary>

**解答:** B、D

拡張イベントセッションをフィルター処理してシステムクエリを除外することで、ユーザーアクティビティにフォーカスが残り、ノイズが減り、分析の効率が向上します。sql_batch_completed イベントのキャプチャは、実行時間の長いクエリやボトルネックを特定するのに役立つ、完了したSQLバッチを監視するために不可欠です。

| 選択肢 | 説明 | 正否 |
|---|---|---|
| A. クエリストアで履歴分析 | 貴重な履歴データを提供するが、リアルタイム監視の要件を満たさない | ❌ |
| B. sql_batch_completed のキャプチャ | 完了したSQLバッチを監視し、実行時間の長いクエリを特定 | ✅ 正解 |
| C. データベース監査の有効化 | 認証の試みや変更を追跡するが、クエリ実行パフォーマンスの分析情報は得られない | ❌ |
| D. 拡張イベントでシステムクエリ除外 | ユーザーアクティビティにフォーカスし分析効率が向上 | ✅ 正解 |

</details>

- B. db_ddladmin

| B. db_ddladmin | データベースに対して任意のDDLコマンドを実行できる。権限昇格のリスクあり | ❌ |
| C. db_owner | データベースのすべての構成およびメンテナンスアクティビティを実行できる | ✅ 正解 |
| D. db_securityadmin | カスタムロールのみのロールメンバーシップの変更および権限の管理のみ | ❌ |

## 問題 5

- B. sys.dm_exec_query_profiles
- C. sys.dm_exec_query_statistics_xml
- D. sys.dm_exec_sessions_wait_stats

| 選択肢 | 説明 | 正否 |
|---|---|---|
| A. sys.dm_db_wait_stats | 操作中に実行されたスレッドによって検出されたすべての待機情報を返す | ❌ |
| B. sys.dm_exec_query_profiles | クエリの実行中にリアルタイムのクエリの進行状況を監視 | ❌ |
| C. sys.dm_exec_query_statistics_xml | 実行中の要求のクエリ実行プランを返す | ❌ |
| D. sys.dm_exec_sessions_wait_stats | セッションごとの待機情報を返す | ✅ 正解 |

</details>

- B. ALTER DATABASE SCOPED CONFIGURATION を使用して、データベースの MAXDOP オプションを構成します。

## 問題 20

組織は、Azure SQL Databaseを使用して顧客の機密情報を格納します。権限のないユーザーが完全なデータを表示できないようにしながら、権限のないユーザーのクエリ結果での機密データの公開を制限する必要があります。ソリューションはアプリケーションに対して透過的であり、既存のクエリに対する最小限の変更が必要です。どの機能を使用する必要がありますか？（一つ選択）

- A. Always Encrypted
- B. 監査
- C. 動的データマスキング
- D. 行レベルセキュリティ

<details>
<summary>解答・解説</summary>

**解答:** C

動的データマスキングは、承認されていないユーザーにマスクされたデータが確実に表示され、承認されたユーザーがマスクされていないデータを表示し、要件に直接対処できるため、最適なソリューションです。

| 選択肢 | 説明 | 正否 |
|---|---|---|
| A. Always Encrypted | 機密データを暗号化するが、動的なマスク機能は提供しない | ❌ |
| B. 監査 | データへのアクセスを追跡・監視するが、機密情報の表示を防ぐことはできない | ❌ |
| C. 動的データマスキング | 承認されたユーザーはマスクなし、未承認ユーザーはマスクされたデータを表示 | ✅ 正解 |
| D. 行レベルセキュリティ | 特定の行へのアクセスを制御するが、機密データをマスクする機能は含まない | ❌ |

</details>
