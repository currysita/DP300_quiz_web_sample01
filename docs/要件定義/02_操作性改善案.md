# 操作性改善案：PCキーボードショートカット

## 背景・課題

スマートフォンでは親指だけで操作できるよう設計されているが、PC からの操作はすべてマウスに依存しており、効率が悪い。問題を次々と解いていく用途では、マウスを動かすコストが積み重なる。

vi（hjkl）や FPS ゲーム（WASD）のように、手をホームポジションから動かさずに操作できるキーバインドを導入する。

---

## 対象画面

キーボードショートカットが有効な画面は **問題画面（quiz.html）のみ**とする。問題集選択画面・結果画面は通常のタブ操作で十分。

---

## キーバインド設計

### 問題ナビゲーション

| キー | 動作 | 備考 |
|---|---|---|
| `→` / `L` / `N` | 次の問題へ | 解答前でも移動可（スキップ）。最終問題では結果画面へ遷移 |
| `←` / `H` / `P` | 前の問題へ | 最初の問題では無効 |

### 選択肢の選択（単一選択・複数選択共通）

| キー | 動作 |
|---|---|
| `A` | 選択肢 A を選択 / トグル |
| `B` | 選択肢 B を選択 / トグル |
| `C` | 選択肢 C を選択 / トグル |
| `D` | 選択肢 D を選択 / トグル |
| `E` | 選択肢 E を選択 / トグル |
| `F` | 選択肢 F を選択 / トグル |

- **単一選択問題**: A〜F キーを押した瞬間に即時採点（現在のクリック動作と同じ）
- **複数選択問題**: A〜F キーで選択をトグル。`Enter` または `Space` で採点ボタンを押す

### 採点

| キー | 動作 | 備考 |
|---|---|---|
| `Enter` / `Space` | 採点する | 複数選択問題で選択数が `answer_count` に達している場合のみ有効 |

### その他

| キー | 動作 |
|---|---|
| `?` | ショートカット一覧を表示（ヘルプ） |

---

## 設計上の注意点

### 解答済み問題でのキー操作

- 解答済みの問題を再訪した場合、A〜F キーおよび `Enter`/`Space` は無効にする（選択肢が `disabled` になっているため、対応する必要はない）
- `→`/`←` によるナビゲーションは解答済みでも有効

### テキストエリアや入力フォームとの競合

- 現在のアプリには入力フォームがないため競合は発生しない
- 将来的に入力要素が追加された場合は、フォーカス中はショートカットを無効にする（`event.target.tagName` で判定）

### ブラウザのデフォルト動作との競合

| キー | ブラウザのデフォルト | 対応 |
|---|---|---|
| `Space` | ページスクロール | `event.preventDefault()` で抑止 |
| `←` `→` | ブラウザの履歴移動（一部ブラウザ） | `event.preventDefault()` で抑止 |
| `N` `P` | 競合なし | そのまま使用 |
| `H` `L` | 競合なし | そのまま使用 |

---

## 実装方針

`document` に `keydown` イベントリスナーを1つ登録し、`quiz.html` のスクリプト内で管理する。

```javascript
$(document).on('keydown', function (e) {
  // 入力フォームへのフォーカス中は無視（将来対応）
  // if (['INPUT', 'TEXTAREA'].includes(e.target.tagName)) return;

  switch (e.key) {
    case 'ArrowRight': case 'l': case 'L': case 'n': case 'N':
      // 次の問題 or 結果画面
      break;
    case 'ArrowLeft': case 'h': case 'H': case 'p': case 'P':
      // 前の問題
      break;
    case 'a': case 'A': triggerChoice('A'); break;
    case 'b': case 'B': triggerChoice('B'); break;
    case 'c': case 'C': triggerChoice('C'); break;
    case 'd': case 'D': triggerChoice('D'); break;
    case 'e': case 'E': triggerChoice('E'); break;
    case 'f': case 'F': triggerChoice('F'); break;
    case 'Enter': case ' ':
      e.preventDefault();
      // 採点ボタンをクリック（複数選択時）
      break;
  }
});
```

## 優先度

| 機能 | 優先度 |
|---|---|
| `→` / `←` による問題ナビゲーション | 高（最も恩恵が大きい） |
| `A`〜`F` キーによる選択肢選択 | 高 |
| `Enter` / `Space` による採点 | 中（複数選択問題で必要） |
| `?` によるヘルプ表示 | 低 |
