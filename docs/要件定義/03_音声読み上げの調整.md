# 音声読み上げ（スクリーンリーダー）対応

## 背景・課題

ブラウザの読み上げ機能（スクリーンリーダー）を使った場合、現状のUIは以下の問題がある。

- `←` `→` などの記号ボタンが「左矢印」「右矢印」と読み上げられる
- バッジ（「問題 3 / 49」「一つ選択」）が視覚的な装飾として読まれ、流れが不自然になる
- 解答後に追加表示される正解・解説が、読み上げ位置を変えないと聞こえない
- 問題番号一覧の小さなボタンが「3」のように数字だけ読まれ、状態（正解・不正解・未回答）が伝わらない
- 上部ナビの `←` `→` ボタンと下部ナビの「← 前の問題」「次の問題 →」が重複して読まれる

---

## 対応方針

WAI-ARIA 属性と HTML のセマンティクスを使い、**コードを変えずに読み上げ体験だけを改善する**。  
視覚的な見た目は変えない。

---

## 対応箇所一覧

### 1. ナビゲーションボタンの `aria-label`

記号だけのボタンに読み上げ用のラベルを付ける。

| 要素 | 現状の読み上げ | 対応後 |
|---|---|---|
| 上部 `←` ボタン | 「左矢印」 | 「前の問題」 |
| 上部 `→` ボタン | 「右矢印」 | 「次の問題」 |
| 下部「← 前の問題」 | 「左矢印 前の問題」 | 「前の問題」（矢印を `aria-hidden` で非表示） |
| 下部「次の問題 →」 | 「次の問題 右矢印」 | 「次の問題」（矢印を `aria-hidden` で非表示） |

```html
<button id="btn-prev-top" aria-label="前の問題">←</button>
<button id="btn-next-top" aria-label="次の問題">→</button>

<button id="btn-prev">
  <span aria-hidden="true">← </span>前の問題
</button>
<button id="btn-next">
  次の問題<span aria-hidden="true"> →</span>
</button>
```

### 2. 問題番号・選択形式バッジを `aria-label` でまとめる

バッジが細切れに読まれるのを防ぐため、バッジを包む `div` に `aria-label` を付け、バッジ自体は `aria-hidden` にする。

```html
<div aria-label="問題 3 / 49、一つ選択">
  <span id="question-index" class="badge bg-secondary" aria-hidden="true"></span>
  <span id="question-type-badge" class="badge bg-primary" aria-hidden="true"></span>
</div>
```

JavaScript 側でバッジのテキストを設定するタイミングで、包む `div` の `aria-label` も同時に更新する。

### 3. 解答後の正解・解説を `aria-live` で自動読み上げ

採点後に表示される結果メッセージと解説を、ユーザーが移動しなくても自動的に読み上げる。

```html
<div id="result-message" role="alert" aria-live="assertive" ...></div>
<div id="explanation-card" aria-live="polite" ...></div>
```

- `aria-live="assertive"` — 正解・不正解メッセージは即座に割り込み読み上げ
- `aria-live="polite"` — 解説カードは現在の読み上げが終わってから読む

### 4. 問題番号一覧を `aria-hidden` で非表示にする

問題番号一覧（`#question-nav`）は補助的なナビであり、問題数が多い場合に全ボタンが読み上げられると冗長になる。一覧全体をスクリーンリーダーから隠す。

現在地と全体数の情報は上部バッジ（§2）が「問題 1 / 20、一つ選択」とまとめて読み上げるため、情報の欠落はない。

```html
<div id="question-nav" class="question-nav-grid" aria-hidden="true"></div>
```

### 5. 選択肢ボタンに選択状態を付ける（複数選択問題）

複数選択問題でトグル選択しているとき、選択中かどうかが読み上げで伝わらない。

```html
<!-- 未選択 -->
<button aria-pressed="false">A. 選択肢テキスト</button>
<!-- 選択中 -->
<button aria-pressed="true">A. 選択肢テキスト</button>
```

- `aria-pressed` はトグルボタンのセマンティクスを持つ
- 採点後は `disabled` になるため `aria-pressed` は不要（`disabled` 状態は自動的に読まれる）

### 6. 廃止警告の読み上げ

`role="alert"` はすでに設定済み。現状のまま対応不要。

---

## 対応しない項目

- 問題番号一覧（`#question-nav`）の各ボタンへの `aria-label` 付与は行わない。一覧全体を `aria-hidden` にするため不要（§4）
- 解説テーブルへの `summary` 属性付与は、現在のスクリーンリーダーでは効果が薄いため見送る

---

## 優先度

| 対応箇所 | 優先度 | 理由 |
|---|---|---|
| ナビボタンの `aria-label` | 高 | 記号だけでは操作不能 |
| `aria-live` による結果読み上げ | 高 | 採点後の結果が聞こえないと使えない |
| 問題番号一覧を `aria-hidden` で隠す | 高 | 全ボタンが読み上げられると冗長。現在地情報は上部バッジで代替 |
| バッジの `aria-label` まとめ | 高 | 現在地・全体数・選択形式を1回でまとめて読むために必要 |
| 選択肢の `aria-pressed` | 中 | 複数選択問題でのみ必要 |
