# 要件定義書

## 1. プロジェクト概要

### 目的
資格試験（DP-300）の学習を支援するため、ブラウザで動作する問題集アプリを作成する。

### 対象ユーザー
- DP-300（Azure Database Administrator Associate）の受験準備をしている学習者
- 主に自分自身で使用する個人学習ツール

### 公開方法
GitHub Pages で静的ファイルとして公開する。サーバーサイド処理は一切行わない。

---

## 2. 機能要件

### 2-1. 問題集選択機能

- 利用可能な問題集の一覧を画面に表示する
- 各問題集には以下の情報を表示する
  - 問題集名
  - 出題範囲（例: 問題 1〜50）
  - 問題数
- 問題集を選択してスタートすると、その問題集の問題画面へ遷移する
- 問題集ファイルは独立した単位とし、複数の問題集を同時に解く機能は持たない

### 2-2. 問題表示機能

- 問題を1問ずつ順番に表示する
- 問題文・選択肢（2〜4択）を表示する
- 問題番号と総問題数を表示する（例: 問題 3 / 49）
- 選択形式は「一つ選択」のみ対応する
- 廃止・出題注意フラグ（`deprecated`）が付いた問題は、問題文の上部に警告を表示する。問題自体は出題する
- スキップフラグ（`skip`）が付いた問題は出題対象から除外し、問題一覧にも表示しない

### 2-3. 採点・解説機能

- 選択肢を押した時点で即時採点する
- 採点後に以下を表示する
  - 正解 / 不正解のメッセージ
  - 正解の選択肢（緑でハイライト）
  - 選択した誤答（赤でハイライト）
  - 解説テキスト
  - 各選択肢の正誤を整理した解説テーブル
- 採点後は選択肢を変更できない

### 2-4. ナビゲーション機能

- 「次の問題」ボタンで次の問題へ進む。採点前は無効とする
- 「前の問題」ボタンで前の問題へ戻ることができる
- 前の問題に戻った場合、解答済みの状態（採点結果・解説）を再表示する
- 最終問題の「次の問題」ボタンは「結果を見る」に変わり、結果画面へ遷移する

### 2-5. 結果表示機能

- 全問終了後に結果画面を表示する
- 表示内容
  - 正解数 / 総問題数
  - 正答率（%）
  - 問題ごとの正誤一覧（不正解の場合は正解ラベルも表示）
- 結果画面から問題集選択画面に戻れる

### 2-6. 学習履歴の記録・表示機能

- 問題ごとの解答結果（正解 / 不正解 / 未回答）を `localStorage` に保存する
- 保存データは問題集単位で管理する
- 問題集選択画面で各問題集の進捗を表示する
  - 解答済み問題数 / 総問題数
  - 正解数・不正解数・未回答数
- 問題画面で各問題の解答済み状態をひと目で確認できるようにする（問題番号一覧など）
- 保存された履歴は同一ブラウザ・同一デバイス上でのみ有効とする
- ブラウザのキャッシュ削除により履歴が消える場合がある（許容する）

### 2-7. 間違えた問題の復習機能

- 問題集選択画面の各カードに「復習」ボタンを表示する
- 復習ボタンは不正解の問題が1問以上ある場合のみ有効とする
- 復習モードでは `localStorage` に記録された不正解の問題のみを出題する
- 復習モードで解答した結果は通常モードと同じく `localStorage` に上書き保存する
- 復習モードであることを問題画面のヘッダーに表示する

### 2-8. 進捗リセット機能

- 問題集選択画面の各カードに「リセット」ボタンを表示する
- リセットボタンは解答済みの問題が1問以上ある場合のみ有効とする
- リセット実行前に確認ダイアログを表示する
- リセットにより該当問題集の `localStorage` データをすべて削除する
- リセット後は進捗表示を「未回答」状態に戻す

---

## 3. 非機能要件

### 3-1. 動作環境
- モダンブラウザ（Chrome / Edge / Firefox 最新版）で動作すること
- スマートフォン・タブレットでも読みやすいレスポンシブレイアウトとする

### 3-2. ホスティング
- GitHub Pages で公開できること
- ビルドツール・サーバーサイド処理を使わず、静的ファイルのみで完結すること

### 3-3. パフォーマンス
- 問題集 JSON の読み込みは `fetch()` で非同期に行う
- ローカルで `file://` プロトコルで開いた場合、`fetch()` が失敗することがある。その場合は Live Server 等を使用する

---

## 4. 問題集データの管理ルール

- 問題集の元データは `問題集の参考資料/` 以下の Markdown ファイルを正とする
- JSON ファイルは Markdown から変換して `data/` に配置する
- 廃止・出題注意の問題は JSON から除外せず `deprecated: true` フラグを付けて残す
- 問題集ファイルを新規追加する場合は、JSON ファイルの配置と `app.js` 内の問題集定義リストへの追記のみで対応できること

---

## 5. 対象外

- ユーザー登録・ログイン機能
- 学習履歴のグラフ・統計表示
- 複数デバイス間での履歴同期
- 問題のランダム出題・出題範囲の絞り込み
- 複数の問題集をまたいだ出題
- 問題の編集・追加 UI
