# 画面設計

## 画面一覧

| 画面 | ファイル | 概要 |
|---|---|---|
| 問題集選択画面 | index.html | 問題集の一覧を表示し、開始する問題集を選ぶ |
| 問題画面 | quiz.html | 問題を1問ずつ表示し、選択・採点・解説を行う |
| 結果画面 | result.html | 全問終了後のスコアと正誤一覧を表示する |

---

## 問題集選択画面（index.html）

### レイアウトイメージ

```
┌─────────────────────────────────┐
│  DP-300 問題集                   │
│  資格試験対策                    │
├─────────────────────────────────┤
│  ┌───────────────────────────┐  │
│  │ 第1回  問題 1〜50          │  │
│  │ 49問                      │  │
│  │ ✅ 32問  ❌ 10問  ─ 7問未回答 │  │  ← localStorage から表示
│  │ [スタート] [復習] [リセット] │  │  ← ボタン群
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │ 第2回  問題 51〜100        │  │
│  │ 50問                      │  │
│  │ ─ 未回答                  │  │  ← 未着手の場合
│  │ [スタート]                 │  │  ← 復習・リセットは無効化
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

### 仕様

- 問題集は `data/` 以下の JSON ファイルを定義リストとして管理（`app.js` 内に記述）
- 各カードに問題集名・問題数を表示
- `localStorage` から各問題集の進捗（正解数・不正解数・未回答数）を読み込んで表示する
- スタートボタン押下で `quiz.html?set=<ファイル名>` へ遷移
- 復習ボタン押下で `quiz.html?set=<id>&mode=review` へ遷移（不正解1問以上の場合のみ有効）
- リセットボタン押下で確認ダイアログを表示し、OKなら `localStorage` データを削除してカードを再描画（解答済み1問以上の場合のみ有効）

## 問題画面（quiz.html）

### レイアウトイメージ

```
┌─────────────────────────────────┐
│ [←] 問題 3 / 49  [一つ選択] [→]│  ← 上部ナビ（解答前後問わず有効）
│ ─────────────────────────────── │
│                                 │
│  問題文がここに表示される。      │
│  複数行にわたることもある。      │
│                                 │
│  [A. 選択肢テキスト           ] │
│  [B. 選択肢テキスト           ] │
│  [C. 選択肢テキスト           ] │
│  [D. 選択肢テキスト           ] │
│                                 │
│ ─────────────────────────────── │
│  ✅ 正解です！                  │  ← 解答後に表示
│                                 │
│  【解答・解説】                  │  ← 解答後に展開
│  解説テキスト…                  │
│  | 選択肢 | 説明 | 正否 |       │
│                                 │
│  [← 前の問題]   [次の問題 →]   │  ← 下部ナビ（「次の問題」は解答後に有効化）
└─────────────────────────────────┘
```

### 仕様

#### 上部ナビと下部ナビの違い

| | 上部ナビ（`[←]` `[→]`） | 下部ナビ（`[← 前の問題]` `[次の問題 →]`） |
|---|---|---|
| 配置 | 問題番号バッジ行の左右 | 解説カードの下 |
| 「次へ」の有効タイミング | 常に有効（解答前スキップ可） | 解答後のみ有効 |
| 最終問題の「次へ」 | 無効（ボタン自体を disabled） | 「結果を見る」に変わり結果画面へ遷移 |
| キーボード対応 | `→` / `←` キーで操作 | 同上（共通のハンドラ） |

- URLパラメータ `set` から問題集JSONを特定し `fetch()` で読み込む
- URLパラメータ `mode=review` が指定された場合は復習モードとして起動する
  - `localStorage` の不正解データを参照し、不正解問題のみを出題する
  - ヘッダーに「復習モード」バッジを表示する
- 問題は配列インデックスで管理し、前後ボタンで移動
- **解答前**: 選択肢ボタンのみ表示。解説は非表示
  - `localStorage` に解答済みデータがある問題は、前回の解答結果を復元して表示する
- **解答後**:
  - 正解の選択肢を緑、誤答を赤でハイライト
  - 正解 / 不正解メッセージを表示
  - 解説を展開表示
  - 「次の問題」ボタンを有効化
  - 解答結果を `localStorage` に保存する
- 廃止フラグ（`deprecated: true`）がある問題は問題文上部に警告バッジを表示
- 最終問題の「次の問題」ボタンは「結果を見る」に変わり `result.html` へ遷移
- 画面下部などに問題番号一覧を表示し、各問の未回答 / 正解 / 不正解を色で区別する

### キーボードショートカット仕様（PC操作性改善）

対象画面: **問題画面（quiz.html）のみ**。`document` に `keydown` イベントリスナーを1つ登録して管理する。

#### ショートカット一覧

| キー | 動作 | 条件 |
|---|---|---|
| `→` / `L` / `N` | 次の問題へ（解答前スキップ可） | 最終問題では無効 |
| `←` / `H` / `P` | 前の問題へ | 最初の問題では無効 |
| `A` | 選択肢 A を選択 / トグル | 解答前のみ有効 |
| `B` | 選択肢 B を選択 / トグル | 解答前のみ有効 |
| `C` | 選択肢 C を選択 / トグル | 解答前のみ有効 |
| `D` | 選択肢 D を選択 / トグル | 解答前のみ有効 |
| `E` | 選択肢 E を選択 / トグル | 解答前のみ有効 |
| `F` | 選択肢 F を選択 / トグル | 解答前のみ有効 |
| `Enter` / `Space` | 採点する | 複数選択問題で選択数が `answer_count` に達している場合のみ |

#### 選択肢キーの動作詳細

- **単一選択問題**: A〜F キーを押した瞬間に即時採点（クリックと同じ動作）
- **複数選択問題**: A〜F キーで選択をトグル。`Enter` または `Space` で採点実行

#### ブラウザデフォルト動作との競合

| キー | ブラウザのデフォルト | 対応 |
|---|---|---|
| `Space` | ページスクロール | `event.preventDefault()` で抑止 |
| `←` `→` | 一部ブラウザで履歴移動 | `event.preventDefault()` で抑止 |

#### 将来対応

入力フォームが追加された場合は、`event.target.tagName` で `INPUT` / `TEXTAREA` を判定してショートカットを無効にする。

### 複数選択問題（type: "multiple"）の仕様

#### 選択形式バッジ

| JSON の type / answer_count | バッジ表示 |
|---|---|
| `single` | `一つ選択` |
| `multiple` / answer_count: 2 | `2つ選択` |
| `multiple` / answer_count: 3 | `3つ選択` |

#### 解答前の操作

```
┌─────────────────────────────────┐
│ 問題 5 / 50          [2つ選択]  │
│ ─────────────────────────────── │
│  問題文...                      │
│                                 │
│  [A. 選択肢テキスト           ] │  ← 未選択（グレー枠）
│  [B. 選択肢テキスト  ✔ 選択中 ] │  ← 選択済（青塗り）
│  [C. 選択肢テキスト           ] │  ← 未選択（グレー枠）
│  [D. 選択肢テキスト  ✔ 選択中 ] │  ← 選択済（青塗り）
│                                 │
│  [採点する]  ← 選択数が answer_count に達したら有効化  │
└─────────────────────────────────┘
```

- 選択肢ボタンはトグル動作（クリックで選択/解除を繰り返す）
- 選択中は `btn-primary`（青塗り）、未選択は `btn-outline-secondary`（グレー枠）で区別
- 選択数が `answer_count` に達したら「採点する」ボタンが有効化される
- 選択数が `answer_count` を超えることはできない（超えるクリックは無視する）
- 「採点する」ボタンは選択数 < answer_count の間は `disabled` 状態

#### 解答後の表示

```
│  [A. 選択肢テキスト  ✔        ] │  ← 正解かつ選択 → 緑
│  [B. 選択肢テキスト           ] │  ← 不正解かつ未選択 → グレー枠（disabled）
│  [C. 選択肢テキスト  ✔ ❌     ] │  ← 不正解かつ選択 → 赤
│  [D. 選択肢テキスト  ✔        ] │  ← 正解かつ未選択 → 緑（正解の表示）
```

- 全選択肢を `disabled` にする
- 正解ラベルに含まれる選択肢 → `btn-success`（緑）
- 正解ラベルに含まれない かつ 選択した選択肢 → `btn-danger`（赤）
- 正解ラベルに含まれない かつ 未選択の選択肢 → `btn-outline-secondary`（グレー枠、disabled）

#### 正誤判定

- 選択ラベルの配列と正解ラベルの配列が**完全一致**するとき正解
  - 順序は問わない（`["A","C"]` と `["C","A"]` は同じ）
  - 過不足があれば不正解

#### 解答結果の localStorage 保存

```json
{
  "answered": true,
  "correct": false,
  "selected": ["B", "D"]
}
```

- `selected` は単一選択と同様に配列で保存する（単一選択は文字列のまま維持）

#### 解答済み問題への再訪

- 前回選択した選択肢をトグル選択済み状態に復元する（青塗り → 採点後ハイライトへ移行した見た目）
- 解答済みの場合は初めから採点後の状態（ハイライト＋解説）で表示する
- 採点ボタンや選択肢クリックはすべて `disabled` にする

#### 問題番号一覧のカラーコード

| 状態 | 色 |
|---|---|
| 未回答 | グレー |
| 正解 | 緑 |
| 不正解 | 赤 |
| 現在表示中 | 青枠 |

### スクリーンリーダー（音声読み上げ）対応

視覚的な見た目は変えず、WAI-ARIA 属性で読み上げ体験を改善する。

#### ナビゲーションボタン

記号だけのボタンは `aria-label` で読み上げ用ラベルを付ける。ボタン内の矢印記号は `aria-hidden="true"` で読み上げから除外する。

| 要素 | 対応 |
|---|---|
| 上部 `←` ボタン | `aria-label="前の問題"` |
| 上部 `→` ボタン | `aria-label="次の問題"` |
| 下部「← 前の問題」 | 矢印部分を `aria-hidden="true"` でスキップ |
| 下部「次の問題 →」 | 矢印部分を `aria-hidden="true"` でスキップ |

#### 問題番号・選択形式バッジ

バッジを包む `div` に `aria-label` を付け、バッジ自体は `aria-hidden="true"` にする。  
JavaScript でバッジテキストを更新するタイミングで `aria-label` も同時に更新する。

```
aria-label の例: "問題 3 / 49、一つ選択"
```

#### 採点結果・解説の自動読み上げ（`aria-live`）

採点後に追加表示される要素を、ユーザーが手動で移動しなくても読み上げる。

| 要素 | 設定 | 理由 |
|---|---|---|
| `#result-message`（正解・不正解） | `aria-live="assertive"` | 即座に割り込み読み上げ |
| `#explanation-card`（解説） | `aria-live="polite"` | 現在の読み上げ終了後に読む |

廃止警告（`#deprecated-alert`）は `role="alert"` が設定済みのため対応不要。

#### 問題番号一覧ボタン（`#question-nav`）

数字だけでは状態が伝わらないため、ボタン生成・更新時に `aria-label` をセットする。

| 状態 | `aria-label` の例 |
|---|---|
| 未回答 | `"問題 3、未回答"` |
| 正解 | `"問題 3、正解"` |
| 不正解 | `"問題 3、不正解"` |
| 現在表示中 | `"問題 3、現在表示中"` |

#### 選択肢ボタンの選択状態（複数選択問題）

複数選択問題でトグル選択中かどうかを `aria-pressed` で伝える。採点後は `disabled` になるため不要。

```html
<button aria-pressed="false">A. 選択肢テキスト</button>  <!-- 未選択 -->
<button aria-pressed="true">A. 選択肢テキスト</button>   <!-- 選択中 -->
```

## 結果画面（result.html）

### レイアウトイメージ

```
┌─────────────────────────────────┐
│  結果                           │
│                                 │
│  38 / 49 問 正解                │
│  正答率: 77%                    │
│                                 │
│  ─────────────────────────────  │
│  問題1  ✅                      │
│  問題2  ✅                      │
│  問題3  ❌  正解: B             │
│  …                              │
│  ─────────────────────────────  │
│                                 │
│       [問題集の選択に戻る]       │
└─────────────────────────────────┘
```

### 仕様

- URLパラメータからスコア・総問題数・各問の正誤を受け取って表示
- 正誤一覧は問題番号・正誤マーク・不正解時は正解ラベルを表示
- 「問題集の選択に戻る」で `index.html` へ遷移
