# 概要設計

## システム概要

資格試験（DP-300）向けのブラウザで動作する問題集アプリ。  
HTML / JavaScript / JSON で構成し、GitHub Pages でそのまま公開できる静的サイト。

---

## ファイル構成

```
/
├── src/
│   ├── index.html              # 問題集選択画面（タイトル画面）
│   ├── quiz.html               # 問題画面
│   ├── result.html             # 結果画面
│   ├── app.js                  # 共通ロジック
│   ├── style.css               # 共通スタイル（Bootstrap 拡張分）
│   └── data/
│       ├── index.json          # 問題集一覧（問題集追加時はここだけ編集）
│       ├── q001-050.json       # 問題集データ（第1回）
│       └── q051-100.json       # 問題集データ（第2回）
├── schema/
│   └── quiz.schema.json        # 問題集JSONのスキーマ定義（開発用）
├── 設計書/
├── 要件定義/
├── 実験用ソース/
└── 問題集の参考資料/            # 元データ（Markdown）編集はここを正とする
```

---

## 画面構成

```
index.html（問題集選択）
    │
    └──► quiz.html（問題画面）
              │
              └──► result.html（結果画面）
                        │
                        └──► index.html（選択画面に戻る）
```

## データの流れ

1. `index.html` 起動時に `data/index.json` を `fetch()` で読み込み、問題集一覧を表示
2. 問題集を選択すると、問題集IDをURLパラメータに付けて `quiz.html` へ遷移  
   例: `quiz.html?set=q001-050`
3. `quiz.html` が対応する JSON を `fetch()` で読み込み、問題を表示
4. 全問終了後、`result.html` へ遷移  
   例: `result.html?set=q001-050`

## 永続化

- セッション中の進捗・スコアはURLパラメータとJavaScript変数で管理
- 問題ごとの解答結果は `localStorage` に保存し、ブラウザを閉じても維持する

### localStorage のデータ構造

キー: `quiz_progress_{問題集ID}`（例: `quiz_progress_q001-050`）

```json
{
  "q001-050": {
    "2": { "answered": true, "correct": true,  "selected": "A" },
    "3": { "answered": true, "correct": false, "selected": "C" },
    "4": { "answered": false }
  }
}
```

- キーは問題の `id`（数値を文字列として使用）
- `answered: false` の問題は未回答扱い
- 問題集選択画面の進捗表示および問題画面の問題番号一覧に使用する
